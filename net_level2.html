Level 2: Network Layers & Protocols

To truly understand how computers talk to each other, we need to study the **layers of communication** and the **rules (protocols)** they follow. These layers help us troubleshoot issues, secure communications, and build more efficient networks.

üìò OSI Model - 7 Layers

The OSI (Open Systems Interconnection) model is a conceptual framework used to understand and standardize how data is sent and received over a network. It breaks down communication into 7 layers, each with its own responsibilities. Here's a closer look:

  1. Physical Layer
    This is the lowest layer and deals with the actual physical hardware ‚Äî wires, cables, switches, and the transmission of bits as electrical or optical signals.
    Example: Ethernet cables, Wi-Fi signals.

  2. Data Link Layer
    This layer is responsible for reliable transmission over the physical link. It organizes bits into frames and handles error detection and correction.
    Example: MAC addresses, Ethernet protocol.
    Analogy: Like traffic rules that control cars on local roads.

  3. Network Layer
    Determines the best path to send data from source to destination across networks. Handles addressing and routing.
    Example: IP (Internet Protocol), Routers.
    Analogy: Like a GPS finding the best route for delivery.

  4. Transport Layer
    Ensures complete data transfer with reliability, flow control, and error recovery.
    Example: TCP (guarantees delivery), UDP (faster, less reliable).
    Analogy: Like choosing between tracked postal delivery (TCP) and regular mail (UDP).

  5. Session Layer
    Manages sessions between two applications. It opens, maintains, and closes connections.
    Example: Remote procedure calls (RPC), NetBIOS.

  6. Presentation Layer
    This layer translates data into a format that the application layer can understand. Handles encryption, compression, and formatting.
    Example: SSL/TLS (encryption), JPEG, ASCII.

  7. Application Layer
    The topmost layer directly used by users via applications like web browsers or email clients.
    Example: HTTP, FTP, DNS, SMTP.

üì° Common Network Protocols (With Use Cases)

Protocols are the rules that define how data is transmitted between devices. Here are some common ones categorized by layer:

  • HTTP/HTTPS (Application Layer): Used for viewing web pages. HTTPS adds encryption for security.
  • FTP (Application Layer): Transfers files between computers.
  • DNS (Application Layer): Translates domain names (like google.com) into IP addresses.
  • SMTP/POP3/IMAP (Application Layer): Used for sending and receiving emails.
  • TCP (Transport Layer): Ensures reliable, ordered delivery of data (e.g., downloading a file).
  • UDP (Transport Layer): Sends data fast but without guarantees (used in video calls or gaming).
  • IP (Network Layer): Addresses and routes data packets across the internet.
  • ICMP (Network Layer): Used for diagnostics (like the ping command).
  • Ethernet/Wi-Fi (Data Link/Physical Layers): Responsible for transmitting data over cables or wirelessly.
